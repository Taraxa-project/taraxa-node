api_ex_handler- if given, is a function that is called when an API method exception is caught. It is supposed to re-throw the current exception internally (using throw;), catch it, and meaningfully convert to RpcServer::Error. It is also allowed to not handle the original exception on re-throw - in which case the default exception handling behavior will be applied by the caller.
