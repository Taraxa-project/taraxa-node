42  // Packets types that are blocked only for processing when received from specific peer & after specific

44  // before it are processed. This blocking dependency is applied only for the same peer so transaction packet from one

50  // This "blocking dependency" is specific just for DagBlockPacket. Ideally only dag blocks with the same level
51  // should be processed. In reality there are situation when node receives dag block with smaller level than the level
52  // of blocks that are already being processed. In such case these blocks with smaller levels can be processed
53  // concurrently with blocks that have higher level. All new dag blocks with higher level than the lowest level from
54  // all the blocks that currently being processed are blocked for processing In this map are saved pairs of <level,
55  // list>, where:
56  // level - What dag level have blocks that are currently being processed
